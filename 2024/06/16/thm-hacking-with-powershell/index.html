0x02 什么是Powershell

Powershell 是使用 .NET 框架构建的 Windows 脚本语言和 shell 环境。

这还允许 Powershell 直接从其 shell 执行 .NET 函数。大多数 Powershell 命令（称为 cmdlet）都是用 .NET 编写的。与其他脚本语言和 shell 环境不同，这些 cmdlet 的输出是对象，这使得 Powershell 在某种程度上是面向对象的。

这也意味着运行 cmdlet 允许您对输出对象执行操作（这使得可以方便地将输出从一个 cmdlet 传递到另一个）。 cmdlet 的正常格式使用动词-名词表示；例如，用于列出命令的 cmdlet 称为 Get-Command

常用的动词包括：

  • Get 得到
  • Start 开始
  • Stop 停止
  • Read 读
  • Write 写
  • New 新的
  • Out 出去

要获取已批准动词的完整列表，请访问此链接。

0x03 基本Powershell命令

现在我们已经了解了 cmdlet 的工作原理 - 让我们探索如何使用它们！这里要记住的主要事情是 Get-Command 和 Get-Help 是你最好的朋友！

使用 Get-Help

Get-Help 显示有关 cmdlet 的信息。要获取特定命令的帮助，请运行以下命令：

Get-Help Command-Name

使用 Get-Command

Get-Command 获取当前计算机上安装的所有 cmdlet。这个 cmdlet 的优点在于它允许进行如下所示的模式匹配

Get-Command Verb-* or Get-Command *-Noun

Object 操作

在上一个任务中，我们了解了每个 cmdlet 的输出如何是一个对象。如果我们想操纵输出，我们需要弄清楚一些事情：

  • 将输出传递给其他 cmdlet
  • 使用特定对象 cmdlet 提取信息

管道符( | ) 用于将输出从一个 cmdlet 传递到另一个 cmdlet。与其他 shell 相比，一个主要区别是 Powershell 将对象传递给下一个 cmdlet，而不是将文本或字符串传递给管道之后的命令。与面向对象框架中的每个对象一样，对象将包含方法和属性。

您可以将方法视为可应用于 cmdlet 输出的函数，并且可以将属性视为 cmdlet 输出中的变量。要查看这些详细信息，请将 cmdlet 的输出传递到 Get-Member cmdlet：

Verb-Noun | Get-Member

运行此命令以查看 Get-Command 的成员的示例是：

Get-Command | Get-Member -MemberType Method

从命令中的上述标志可以看出，您还可以在方法和属性之间进行选择。

从之前的 cmdlet 创建对象

操作对象的一种方法是从 cmdlet 的输出中提取属性并创建新对象。这是使用 Select-Object cmdlet 完成的。

您还可以使用以下标志来选择特定信息：

  • first - 获取第一个 x 对象
  • last - 获取最后一个 x 对象
  • unique - 显示独特的对象
  • skip - 跳过 x 个对象

过滤对象

检索输出对象时，您可能希望选择与非常特定的值匹配的对象。您可以使用 Where-Object 根据属性值进行过滤来执行此操作。

使用此 cmdlet 的一般格式是

Verb-Noun | Where-Object -Property PropertyName -operator Value
Verb-Noun | Where-Object {$_.PropertyName -operator Value}

第二个版本使用 $_ 运算符迭代传递给 Where-Object cmdlet 的每个对象。

Powershell 非常敏感，所以不要在命令两边加上引号！

其中 -operator 是以下运算符的列表：

  • -CContains ：如果属性值中的任何一项与指定值完全匹配
  • -EQ ：如果属性值与指定值相同
  • -GT ：如果属性值大于指定值

有关operators的完整列表，请使用此链接。

当 cmdlet 输出大量信息时，您可能需要对其进行排序以更有效地提取信息。您可以通过将 cmdlet 的输出通过管道传输到 Sort-Object cmdlet 来完成此操作。

命令的格式为：

命令练习

练习一：文件“interesting-file.txt”的位置是什么

Get-ChildItem -Path C:\ -Include *interesting-file.* -Recurse
//标准：Get-ChildItem -Path C:\ -Include *.txt -File -Recurse -ErrorAction SilentlyContinue

练习二：指定该文件的内容

Get-Content "C:\Program Files\interesting-file.txt.txt"
//标准：Get-Content -Path "C:\Program Files\interesting-file.txt.txt"

练习三：系统上安装了多少个 cmdlet（仅 cmdlet，不包括函数和别名）？

$cmdlets = Get-Command -CommandType Cmdlet
$cmdlets.Count
//标准：Get-Command | Where-Object -Property CommandType -eq Cmdlet | Measure-Object

练习四：获取interesting-file.txt的MD5哈希值

Get-ChildItem -Path C:\ -Include *interesting-file.* -Recurse | Get-FileHash -Algorithm MD5

练习五：获取当前工作目录的命令是什么？

Get-Location

练习六：路径“C:\Users\Administrator\Documents\Passwords”是否存在（Y/N）？

Test-Path "C:\Users\Administrator\Documents\Passwords"

练习七：您将使用什么命令向 Web 服务器发出请求？

Invoke-WebRequest or Invoke-RestMethod

练习八：在 Windows 上对文件 b64.txt 进行 Base64 解码。

Get-ChildItem -Path C:\ -Include b64.txt //得到： C:\Users\Administrator\Desktop
$base64Content = Get-Content -Path "C:\Users\Administrator\Desktop\b64.txt"
$bytes = [System.Convert]::FromBase64String($base64Content)
[System.Text.Encoding]::UTF8.GetString($bytes)

0x04 枚举

当您获得对任何机器的初始访问权限时，第一步是枚举。我们将列举以下内容：

  • users
  • basic networking information
    基本网络信息
  • file permissions 文件权限
  • registry permissions 注册表权限
  • 计划任务和运行任务
  • insecure files 不安全的文件

您的任务是回答以下问题，以使用 Powershell 命令枚举计算机！

枚举练习

练习一：机器上有多少用户？

(Get-LocalUser).count

练习二：这个SID(S-1-5-21-1394777289-3961777894-1791813945-501)属于哪个本地用户？

Get-LocalUser -SID S-1-5-21-1394777289-3961777894-1791813945-501

练习三：有多少用户将其密码所需值设置为 False？

参考：

(Get-LocalUser | Select-Object * | Where-Object {$_.PasswordRequired -eq $False}).count

练习四：存在多少个本地团体？

(Get-LocalGroup).count

练习五：你用什么命令来获取IP地址信息？

Get-NetIPAddress

练习六：有多少个端口被列为正在侦听？

(Get-NetTCPConnection | Where-Object -Property State -eq "Listen").count

练习七：监听445端口的本地端口的远程地址是多少？

Get-NetTCPConnection | Where-Object {($_.State -eq "Listen") -and ($_.LocalPort -eq "445")}

练习八：已经应用了多少个补丁？

(Get-HotFix).count

练习九：ID KB4023834 的补丁是什么时候安装的？

Get-HotFix | Where-Object {$_.HotFixID -eq "KB4023834"}

练习十：查找备份文件的内容。

Get-ChildItem “*.bak*” -Path C:\ -Recurse | Get-Content

练习十一：搜索所有包含API_KEY的文件

Get-ChildItem -Path "C:\" -Recurse | Select-String -Pattern "API_KEY"

练习十二：您使用什么命令来列出所有正在运行的进程？

Get-Process

练习十三：名为new-sched-task的计划任务的路径是什么？

Get-ScheduledTask | Where-Object {$_.TaskName -eq "new-sched-task"}

练习十四：谁是 C:\ 的所有者

Get-Acl -Path "C:\"

0x05 基本脚本挑战

现在我们已经运行了 Powershell 命令，让我们尝试编写并运行脚本来执行更复杂和更强大的操作。

对于这个问题，我们将使用 Powershell ISE（Powershell 文本编辑器）。让我们使用一个特定的场景来展示此脚本的示例。给定一个端口号列表，我们希望使用该列表来查看本地端口是否正在侦听。使用 Powershell ISE 打开桌面上的 waiting-ports.ps1 脚本。 Powershell 脚本通常具有 .ps1 文件扩展名。

$system_ports = Get-NetTCPConnection -State Listen

$text_port = Get-Content -Path C:\Users\Administrator\Desktop\ports.txt

foreach($port in $text_port){

    if($port -in $system_ports.LocalPort){
        echo $port
     }

}

在第一行，我们想要获取系统上正在侦听的所有端口的列表。我们使用 Get-NetTCPConnection cmdlet 来执行此操作。然后，我们将此 cmdlet 的输出保存到变量中。创建变量的约定如下：

$variable_name = value

在下面的行中，我们要从文件中读取端口列表。我们使用 Get-Content cmdlet 来执行此操作。同样，我们将此输出存储在变量中。最简单的下一步是迭代文件中的所有端口以查看端口是否正在侦听。要迭代文件中的端口，我们使用以下命令：

foreach($new_var in $existing_var){}

这个特定的代码块用于循环访问一组对象。一旦我们有了每个单独的端口，我们想要检查该端口是否出现在侦听本地端口中。我们不再执行另一个 for 循环，而是使用带有 -in 运算符的 if 语句来检查端口是否存在于任何对象的 LocalPort 属性中。可以在此处找到 if 语句比较运算符的完整列表。要运行脚本，请使用 Powershell 调用脚本路径或单击 Powershell ISE 上的绿色按钮：

0x06 中级脚本编写

现在您已经了解了一些脚本的工作原理 - 让我们尝试一些更有趣的事情。有时我们可能没有像 Nmap 和 Python 这样的实用程序可用，我们被迫编写脚本来完成非常基本的任务。

为什么不尝试使用 Powershell 编写一个简单的端口扫描器？以下是一般使用方法：

  • 确定要扫描的 IP 范围（在本例中为 localhost），您可以按照您想要的任何方式提供输入
  • 确定要扫描的端口范围
  • 确定要运行的扫描类型（在本例中它将是简单的 TCP 连接扫描）
$TargetIP = "localhost"
$StartPort = 130
$EndPort = 140

for ($port = $StartPort; $port -le $EndPort; $port++){
	if((Test-NetConnection -ErrorAction SilentlyContinue $TargetIP -Port $port).PingSucceeded -or (Test-NetConnection -ErrorAction SilentlyContinue $TargetIP -Port $port).TcpTestSucceeded){
		$port | Out-File .\OpenPort.txt -Append
		Write-Host "Port $port is open on $TargetIP"
	}
	else{
		Write-Host "Port $port is closed on $TargetIP"
	}
}

(Get-Content .\OpenPort.txt).Count
