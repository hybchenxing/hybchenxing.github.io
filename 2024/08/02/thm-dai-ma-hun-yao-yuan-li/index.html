利用与工具无关的软件混淆实践来隐藏恶意功能并创建独特的代码。

0x01 简介

混淆是针对检测技术的规避方法以及阻碍研究人员进行恶意软件分析的重要组成部分。混淆最初是为了保护软件和知识产权不被窃取或复制，时至今日，虽然它仍然被广泛用于其最初目的，但是网络攻击者已经在将混淆用于恶意目的。

在本文内容中，我们将从多个角度观察和分解混淆方法。

本文的学习目标

  1. 了解如何使用与特定工具无关的混淆来规避现代检测工程；
  2. 理解混淆的原理及其在知识产权保护方面的起源；
  3. 实施混淆方法来隐藏恶意函数。

在开始学习本文内容之前，建议先熟悉基本的编程逻辑和语法，以及了解C、PowerShell(但这不是必需的)。

0x02 混淆的起源

混淆被广泛应用于许多与软件相关的领域，以保护IP(Intellectual Property-知识产权)和应用程序可能包含的其他专有信息。

例如，流行游戏Minecraft(我的世界)就使用了混淆器ProGuard来混淆和最小化其Java类。Minecraft还发布了带有有限信息的混淆映射(maps)，作为旧的未混淆类和新的混淆类之间的转换器，以支持mod社区。

0x03 混淆的静态规避功能

反病毒引擎和EDR(端点检测和响应)解决方案是攻击者所要面临的两个相当重要的安全边界，它们都将利用一个具备大量已知签名的数据库，这些已知签名包括了静态签名和启发式签名(与应用程序行为相关)。

为了规避基于签名的检测，攻击者可以尝试利用广泛的逻辑和语法规则来实施混淆。攻击者通常会通过滥用数据混淆实践来实现其恶意目的，这些实践将在合法的应用程序中隐藏一些重要的可识别信息，上文提到的分层混淆分类法在code-element layer(代码元素层)下很好地总结了这些实践.

下面是obfuscating data(混淆数据)子层中由分层混淆分类法所涵盖的方法表：

Obfuscation Method(混淆方法) Purpose(目的)
Array Transformation(数组变换) 通过拆分、合并、折叠和展平来变换数组
Data Encoding(数据编码) 用数学函数或密码算法对数据进行编码
Data Procedurization(数据程序化/数据加工处理) 使用过程调用替换静态数据
Data Splitting/Merging(数据拆分/合并) 将一个变量中的信息分配到多个新变量中

在下文中，我们将主要关注数据拆分/合并，由于静态签名的防护力度较弱，我们在进行初步混淆时，通常只需要关注这一个方面即可。

0x04 对象连接(串联)

连接(串联)是一种常见的编程概念，它是指将两个单独的对象组合成一个对象，例如字符串。

预定义的运算符定义了在何处进行串联以组合两个独立的对象，下面是Python中字符串连接的简单示例：

>>> A = "Hello "
>>> B = "THM"
>>> C = A + B
>>> print(C)
Hello THM
>>>

根据程序中所使用的具体编程语言，可能有不同或多个可用于连接的预定义操作符，下面是关于一些常用编程语言及其相应的预定义操作符的简单表格。

根据程序中所使用的具体编程语言，可能有不同或多个可用于连接的预定义操作符，下面是关于一些常用编程语言及其相应的预定义操作符的简单表格。

Language(语言) Concatenation Operator(连接运算符)
Python “+”
PowerShell “+”, ”,”, ”$”, 或者根本没有操作符
C# “+”, “String.Join”, “String.Concat”
C “strcat”
C++ “+”, “append”

对象连接这个概念对于攻击者而言意味着什么呢？连接可以为多个攻击向量打开大门，从而让攻击者能够修改签名或者操纵应用程序的其他方面。恶意软件中最常见的对象连接示例是破坏目标静态签名，攻击者可以使用对象连接来先发制人地破坏程序的所有对象，并试图在不追踪它们的情况下立即删除所有签名。

下面我们将观察一个静态的Yara规则，并尝试使用对象连接(串联)来规避静态签名检测。

rule ExampleRule
{
    strings:
        $text_string = "AmsiScanBuffer"
        $hex_string = { B8 57 00 07 80 C3 }

    condition:
        $my_text_string or $my_hex_string
}

这是一个YARA规则的示例，YARA是一种用于检测恶意软件特征的工具，可以通过定义规则来识别恶意文件中的特定字符串、十六进制值等内容；在这个示例规则中，定义了两个字符串匹配项，一个是文本字符串"AmsiScanBuffer"，另一个是十六进制字符串{ B8 57 00 07 80 C3 }，上述规则的条件部分指定了只要匹配任何一个规定的字符串即可触发YARA示例规则的检测机制。

当使用Yara扫描已编译的二进制文件时，如果存在已定义的字符串，它将产生积极的警报/检测信息；如果我们进行对象串联处理，那么就能够让字符串在功能上相同，但是在被扫描时将显示为两个独立的字符串，从而不会产生相关的警报。

IntPtr ASBPtr = GetProcAddress(TargetDLL, "AmsiScanBuffer"); 
↓
IntPtr ASBPtr = GetProcAddress(TargetDLL, "Amsi" + "Scan" + "Buffer");

0x05 混淆的分析欺骗功能

在混淆了恶意代码的基本函数之后，它可能能够通过软件安全检测，但仍然容易受到人为分析的影响，因为分析师和逆向工程师可以深入了解恶意应用程序的函数并停止其相关的恶意操作。

为此，攻击者可以利用先进的逻辑和数学技术来创建更复杂和更难以理解的代码，以对抗分析和逆向工程。

上文提到的分层混淆分类法在code-element layer(代码元素层)的其他子层下很好地总结了这些实践。

下面是混淆布局(obfuscating layout)和混淆控制子层(obfuscating controls sub-layers)中的分类所涵盖的方法表。

Obfuscation Method(混淆法) Purpose(目的)
Junk Code(垃圾代码) 添加无用的垃圾指令，也被称为代码存根
Separation of Related Code(分离相关代码) 将相关代码或指令分离，以增加程序的阅读难度
Stripping Redundant Symbols(去除冗余符号) 剥离多余的符号信息，例如调试(debug)信息或其他符号表
Meaningless Identifiers(无意义的标识符) 将有意义的标识符转换为无意义的标识符
Implicit Controls(隐式控制) 将显式控制指令转换为隐式指令
Dispatcher-based Controls(基于调度程序的控制) 确定在运行时期间将被执行的下一个块
Probabilistic Control Flows(概率性的控制流) 引入具有相同语义但不同语法的控制流复制
Bogus Control Flows(虚假的控制流) 故意添加到程序中但永远不会被执行的控制流

在下文内容中，我们将简单演示上表中的一些混淆方法。

0x06 代码流程和逻辑

控制流(Control flow)是程序执行的关键组成部分，它将定义程序如何符合逻辑地进行。逻辑(Logic)是应用程序的控制流最重要的决定因素之一，包含各种用法，例如if/else语句、for循环等等。传统上，程序是自上而下执行的，当遇到逻辑语句时，则会按照该语句继续执行。

下表列出了在处理控制流或程序逻辑时可能会遇到的一些逻辑语句。

Logic Statement(逻辑语句) Purpose(目的)
if/else 仅(if)当满足条件时才执行，否则(else)将执行其他不同代码块
try/catch 如果程序无法处理错误，将尝试(try)执行代码块并捕获(catch)错误
switch case switch将遵循与if语句类似的条件逻辑，但在解析为中断(break)或默认(default)之前，会使用case检查几种不同的可能条件
for/while loop for循环将基于一定数量的条件进行执行；while循环将持续执行直到不再满足既定条件

为了使概念具体化，我们可以观察以下示例函数及其相应的CFG(Control Flow Graph-控制流图示)来了解该函数可能存在的控制流路径。

x = 10 
if(x > 7):
	print("This executes")
else:
	print("This is ignored")

分析人员可以尝试通过控制流来理解程序的功能，而攻击者则可以尝试操纵程序的逻辑和控制流程来实现攻击目的。在实际处理程序的控制流时，攻击者的目标往往是引入足够多的晦涩且任意的逻辑来迷惑分析人员，但同时又需要避免引入太多的逻辑使得相关程序被AV解决方案检测为恶意。

在下文内容中，我们将简单讨论攻击者可以用来迷惑分析师的不同控制流模式。

0x07 任意控制流模式

为了制作任意的控制流模式，我们可以利用数学、逻辑和其他复杂算法将不同的控制流注入到恶意函数中。

我们可以利用谓词来制作这些复杂的逻辑和/或数学算法。谓词指的是关于输入函数返回True或者False的决策，如果从更高的层次上来分析这个概念，我们可以把谓词想象成一个类似于if语句用于确定代码块是否执行的条件，如上一小节中的示例所示。

谓词的概念也可以被应用于混淆技术中，我们可以使用不透明谓词来控制已知的输出和输入。

论文《不透明谓词：被混淆的二进制代码中的攻击和防御》指出——“不透明谓词是指混淆器知道其值但仍然难以推断的谓词，它可以无缝地与其他混淆方法(如垃圾代码)一起应用，从而将逆向工程的尝试变成一项艰巨的工作”。不透明谓词属于分层混淆分类法中的虚假控制流和概率控制流方法，它们可以被用于向程序中任意添加逻辑或者重构已有函数的控制流。

学习不透明谓词这一主题需要我们对数学和计算原理有更加深入的理解，因此我们在此不进行深入讨论，但我们将简单介绍一个常见的示例。

Collatz(考拉兹)猜想是一个常见的数学问题，它可以被用作不透明谓词的简单示例，此猜想规定：对于任何正整数，重复两个简单的算术运算，最终会将每个正整数都变成1。该数学猜想认为，无论初始的正整数值是多少，经过规定的运算过程后，最终得到的值都会收敛到1。

Collatz猜想：对于任何正整数n，如果n是偶数，则将n除以2；如果n是奇数，则将n乘以 3再加上1；重复这个过程，最终都会得到1。

有关Collatz猜想的更多信息，请参考：https://mathworld.wolfram.com/CollatzProblem.html

事实上，我们知道如果对于一个已知的输入(一个正整数)，它总是会输出1，这意味着它是一个可行的不透明谓词。

下面是一个在Python中应用Collatz猜想的示例。

x = 0
while(x > 1):
	if(x%2==1):
		x=x*3+1
	else:
		x=x/2
	if(x==1):
		print("hello!")

在上面的代码片段中，Collatz猜想仅在x > 1时执行其规定的数学运算，得到的结果为1或TRUE。根据Collatz问题的定义，对于一个正整数输入，它最后将始终返回1，因此如果x是大于1的正整数，则该语句将始终返回true。

可识别信息可能是分析人员可以用来剖析和尝试理解恶意程序的最关键的组件之一。通过限制可识别信息(变量、函数名等)的数量，分析人员可以使得攻击者更有可能无法重建其原始函数。

在较高的层次上，我们应该考虑三种不同类型的可识别数据：代码结构、对象名称和文件/编译属性。在本小节中，我们将分解每个相关的核心概念，并对每个相关的实际方法进行案例研究。

对象名称

对象名称提供了对程序功能的一些最重要的深刻见解，并且可以揭示一个函数的确切用途。虽然分析人员还可以从函数的行为中解构函数的用途，但如果没有函数的上下文，这就会变得更加困难。

文字对象名称的重要性可能会根据相关的计算机语言是编译性的还是解释性的而发生变化。如果使用Python或PowerShell等解释性语言，则所有对象都很重要并且必须进行修改；如果使用诸如C或C#之类的编译性语言，则通常只有出现在字符串中的对象才有意义，对象可以通过任何产生IO操作的函数出现在字符串中。

代码结构

对于恶意软件开发者而言，在处理恶意代码的各个方面时，代码结构可能会是一个令人烦恼的问题，因为它通常会被忽视且不易被识别。如果恶意程序的代码结构在解释型语言和编译型语言中都没有得到充分处理，那么就可能会导致此程序存在可识别的签名或者使得分析人员更容易进行逆向工程。

在可疑程序未被编译的情况下，分析人员可以更深入地了解可疑程序，并且如果没有人为地夸大复杂性，那么分析人员就可以专注于应用程序的确切恶意函数；但正如前面提到的分层混淆分类法论文中所述，垃圾代码和重新排序的代码都可以被广泛地用作附加措施来增加解释型程序的复杂性(这就意味着改变原代码结构)。

此外，我们也可以通过分离相关代码来尝试影响解释型语言和编译型语言所编写的恶意程序的代码结构，这可能导致出现难以识别的隐藏签名。我们知道，启发式签名引擎可以根据目标环境中的函数或 API 调用来确定相关程序是否属于恶意，而为了规避这些签名检测技术，攻击者就可以将相关代码的出现随机化，以欺骗检测引擎，使其相信这是一个安全的调用或安全的函数。

文件&编译属性

已编译的二进制文件的一些次要方面，例如编译方法，可能看起来并不是关键组件，但它们仍然可以带来一些优势以帮助分析人员。例如，如果一个程序被编译为调试版本(debug build)，那么分析人员就可以尝试获得该程序的所有可用的全局变量和其他程序信息。

当程序被编译为调试版本时，编译器将包含一个符号文件。符号通常有助于调试二进制映像，并且可以包含全局和局部变量、函数名称、入口点。攻击者必须意识到这些可能存在的问题，以确保恶意代码有正确的编译实践，从而不会将重要信息泄露给恶意软件分析人员。

幸运的是，对于攻击者而言，符号文件可以被轻松地删除。要从Visual Studio等编译器中删除符号，我们需要先将编译目标从Debug更改为Release，或者我们也可以选择使用像mingw这样的轻量级编译器来对恶意代码进行编译。

如果我们想要从预编译映像中删除符号，那么我们可以使用命令行实用程序：strip。

注意：前面提到的分层混淆分类法白皮书在代码元素(code-element)层下的"剥离冗余符号"方法中很好地总结了怎样处理符号。
