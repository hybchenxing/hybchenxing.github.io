THM-红队威胁Intel

0x01 介绍

威胁情报 (TI) 或网络威胁情报 (CTI) 是归因于对手的信息或 TTP（战术、技术和程序），防御者通常使用它来帮助采取检测措施。红细胞可以从进攻角度利用 CTI 来协助对手模拟。

学习目标

  • 了解威胁情报的基础知识以及如何将其应用于红队活动。
  • 了解如何创建威胁情报驱动的活动。
  • 使用框架来理解概念并利用威胁情报。

0x02 什么是威胁情报

扩展任务 1，可以通过收集由 ISAC（信息和共享分析中心）共同分发和维护的 IOC（妥协指标）和 TTP 来使用 CTI（对数据采取行动）。情报平台和框架也有助于 CTI 的消耗，主要关注所有活动的总体时间表。

注意：ISAC 一词在威胁情报领域中广泛使用，通常指威胁情报平台。

传统上，防御者使用威胁情报为不断变化的威胁形势提供背景信息并量化结果。 IOC 通过对手留下的痕迹（例如域、IP、文件、字符串等）进行量化。蓝队可以利用各种 IOC 来构建检测和分析行为。从红队的角度来看，您可以将威胁情报视为红队对蓝队正确利用 CTI 进行检测的能力的分析。

在这个房间中，我们将重点关注 APT（高级持续威胁）活动以及如何利用其记录的 TTP。下一个任务将详细介绍威胁情报的细节及其对红队的意义。

0x03 威胁情报应用于红队

如前所述，红队将利用 CTI 来帮助对手模拟并支持对手行为的证据。

为了帮助消耗 CTI 和收集 TTP，红队通常会使用威胁情报平台和框架，例如 MITRE ATT&CK、TIBER-EU 和 OST Map。

这些网络框架将收集已知的 TTP，并根据不同的特征对它们进行分类，例如：

  1. Threat Group 威胁组织
  2. Kill Chain Phase 杀伤链阶段
  3. Tactic 战术
  4. Objective/Goal 目的/目标

一旦选择了目标对手，目标就是识别属于该选定对手的所有 TTP，并将它们映射到已知的网络杀伤链。下一个任务将进一步介绍这个概念。

利用 TTP 是一种规划技术，而不是团队在参与执行期间关注的重点。根据团队的规模，可能会雇用 CTI 团队或威胁情报操作员来为红队收集 TTP。在执行交战过程中，红队将使用威胁情报来制作工具、修改流量和行为，并模仿目标对手。任务 5 进一步介绍了这一概念。

总体而言，红队利用威胁情报，通过收集的 TTP 和 IOC 来分析和模拟对手的行为。

0x04 TIBER-EU框架

TIBER-EU（基于威胁情报的道德红队）是欧洲中央银行开发的一个通用框架，以威胁情报的使用为中心。

欧洲央行 TIBER-EU 白皮书中指出，“基于威胁情报的道德红队框架 (TIBER-EU) 使欧洲和国家当局能够与金融基础设施和机构（以下统称为“实体”）合作，制定一个计划来测试和提高他们抵御复杂网络攻击的能力。”

该框架与其他框架的主要区别在于“测试”阶段，需要威胁情报来为红队的测试提供支持。

该框架包含最佳实践，而不是从红队角度来看任何可操作的内容。

如果您有兴趣进一步阅读此框架，有几份公开白皮书和文档，

  • https://www.ecb.europa.eu/pub/pdf/other/ecb.tiber_eu_framework.en.pdf
  • https://www.crest-approved.org/membership/tiber-eu/

0x05 TTP 映射

红细胞使用 TTP 映射将对手收集的 TTP 映射到标准网络杀伤链。将 TTP 映射到杀伤链有助于红队规划模拟对手的交战。

要开始映射 TTP 的过程，必须选择一个对手作为目标。可以根据以下条件选择对手：

  1. Target Industry 目标行业
  2. Employed Attack Vectors 使用攻击向量
  3. Country of Origin 原产地
  4. Other Factors 其他因素

作为此任务的一个示例，我们决定使用 APT 39，这是一个由伊朗部委运营的网络间谍组织，以针对多种行业而闻名。

我们将使用洛克希德·马丁公司的网络杀伤链作为我们的标准网络杀伤链来映射 TTP。

我们将从 MITRE ATT&CK 收集 TTP 的第一个网络框架。如果您不熟悉 MITRE ATT&CK，它提供了分类 TTP 的 ID 和描述。有关 MITRE 以及如何使用 ATT&CK 的更多信息，请查看 MITRE 房间。

ATT&CK 提供了小组收集的 TTP 的基本摘要。我们可以使用 ATT&CK Navigator 来帮助我们可视化每个 TTP 并对其在杀伤链中的位置进行分类。 Navigator 可视化 ATT&CK 链，并在相应的子部分下突出显示对手指定的 TTP。

要使用 ATT&CK 导航器：导航到组摘要页面，在“使用的技术”旁边，导航到“ATT&CK 导航器层”，从下拉菜单导航到“视图”。 ATT&CK 导航器层应该已打开，并且所选组的 TTP 在新选项卡中突出显示。

通过导航器层，我们可以分配我们想要在参与过程中使用的各种 TTP。下面是一个已编译的杀伤链，其中包含 APT39 的映射 TTP。

  1. Reconnaissance: 没有确定的 TTP，使用内部团队方法

  2. Weaponization:

    命令和脚本解释器

    • PowerShell 电源外壳

    • Python

    • VBA 编程语言

    • User executed malicious attachments
      用户执行恶意附件

  3. Delivery:

    • Exploit Public-Facing Applications
      利用面向公众的应用程序
    • Spearphishing 鱼叉式网络钓鱼

  4. Exploitation:

    • Registry modification 注册表修改
    • Scheduled tasks 计划任务
    • Keylogging 键盘记录
    • Credential dumping 凭证倾销

  5. Installation:

    • Ingress tool transfer Ingress 工具传输
    • Proxy usage 代理使用

  6. Command & Control:

    • Web protocols (HTTP/HTTPS)
      网络协议 (HTTP/HTTPS)
    • DNS 域名系统

  7. Actions on Objectives

    • Exfiltration over C2 通过 C2 泄露

MITRE ATT&CK 将完成大部分所需的工作，但我们也可以通过其他平台和框架补充威胁情报信息。 TTP 框架的另一个示例是 OST Map。

OST 地图提供了一个可视化地图来链接多个威胁参与者及其 TTP。

其他开源和企业威胁情报平台可以帮助红队人员进行对手模拟和 TTP 映射，例如：

  • Mandiant Advantage
  • Ontic
  • CrowdStrike Falcon CrowStrike

答案来源为：https://mitre-attack.github.io/attack-navigator//#layerURL=https%3A%2F%2Fattack.mitre.org%2Fgroups%2FG0008%2FG0008-enterprise-layer.json

0x06 CTI的其他红队应用

CTI 还可以在交战执行期间使用，模拟对手的行为特征，例如

  • C2 Traffic
    • User Agents 用户代理
    • Ports, Protocols 端口、协议
    • Listener Profiles 听众资料
  • Malware and Tooling
    • IOCs 国际奥委会
    • Behaviors 行为

我们将展示的 CTI 的第一个行为用途是 C2（命令与控制）流量操纵。红队可以使用 CTI 来识别对手的流量并修改其 C2 流量以进行模拟。

红队根据收集的 CTI 修改 C2 流量的一个示例是可延展的配置文件。可延展的配置文件允许红队操作员控制 C2 侦听器流量的多个方面。

配置文件中要实现的信息可以从 ISAC 和收集的 IOC 或数据包捕获中收集，包括：

  • Host Headers
  • POST URIs
  • Server Responses and Headers

收集到的流量可以帮助红队使他们的流量看起来与目标对手相似，以更接近对手模拟的目标。

CTI 的第二个行为用途是分析对手恶意软件和工具的行为和操作，以开发模拟类似行为或具有类似重要指标的攻击性工具。

一个例子是对手使用自定义滴管。红队可以通过以下方式模仿滴管：

  • Identifying traffic 识别流量
  • Observing syscalls and API calls
    观察系统调用和 API 调用
  • Identifying overall dropper behavior and objective
    识别整体滴管行为和目标
  • Tampering with file signatures and IOCs
    篡改文件签名和 IOC

从行为威胁情报中收集的情报和工具可以帮助红队准备用于执行计划的 TTP 的特定工具。

0x07 创建威胁Interl驱动的任务

威胁情报驱动的活动将利用之前涵盖的所有知识和主题，并将它们结合起来以创建精心策划和研究的活动。

这个房间中的任务流程在逻辑上遵循与红队开始规划活动相同的路径，

  1. 识别框架和一般杀伤链
  2. 确定目标对手
  3. 识别对手的 TTP 和 IOC
  4. 将收集到的威胁情报映射到杀伤链或框架
  5. 起草和维护所需的参与文档
  6. 确定并使用所需的参与资源（工具、C2 修改、域等）

在此任务中，我们将逐步了解红队在规划威胁情报驱动的活动的整个过程中的思维过程。

规划威胁情报驱动的活动最困难的部分可能是映射两个不同的网络框架。为了使这个过程更简单，我们提供了一个基本表格来比较洛克希德马丁网络杀伤链和 MITRE ATT&CK 框架。

Weaponization Execution
Delivery Initial Access
Exploitation Initial Access
Installation Persistence / Defense Evasion

0x08 结论

在规划交战时，必须注意对手模拟的重要性以及威胁情报如何帮助您识别对手及其行为。

每个红队都有自己收集和消化现实世界威胁情报的方法。这个房间涵盖了通常应用于红队场景的不同概念的基本知识。

在计划交战时，请记住，从各个角度看待场景至关重要：进攻、防守和对手。

威胁情报使我们作为红队能够利用蓝队的方法来更深入地了解对手的行为。
