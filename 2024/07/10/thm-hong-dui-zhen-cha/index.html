THM-红队侦察

hybcx2024-07-102024-08-01

0x01 介绍

“了解你的敌人，了解他的剑。”宫本武藏在他的书《五环之书：经典战略指南》中写道。他还写道：“你可以通过了解敌人的时机并利用敌人意想不到的时机来赢得战斗。”虽然这是在剑与矛赢得战斗时写下的，但它也适用于网络空间，在网络空间中，攻击是通过键盘和精心设计的数据包发起的。您对目标的基础设施和人员了解得越多，就越能更好地策划攻击。

在红队行动中，您可能只从公司名称开始，您需要从该公司名称开始收集有关目标的信息。这就是侦察发挥作用的地方。侦察（recon）可以定义为对您的目标（客户）进行初步调查或观察，而不提醒他们注意您的活动。如果你的侦察活动产生太多噪音，对方就会收到警报，这可能会降低你成功的可能性。

该房间的任务涵盖以下主题：

  • Types of reconnaissance activities
    侦察活动的类型
  • WHOIS and DNS-based reconnaissance
  • Advanced searching 高级搜索
  • Searching by image 按图搜索
  • Google Hacking
  • Specialized search engines
    专门的搜索引擎
  • Recon-ng
  • Maltego

我们将讨论的一些具体目标包括：

  • 发现与我们的目标公司相关的子域
  • 收集有关主机和 IP 地址的公开信息
  • 查找与目标相关的电子邮件地址
  • 发现登录凭据和泄露的密码
  • 找到泄露的文档和电子表格

侦察可以分为两部分——被动侦察和主动侦察，如任务 2 中所述。在这个房间中，我们将重点关注被动侦察，即不向目标发出警报或产生“噪音”的技术。在后面的房间中，我们将使用主动侦察工具，这些工具本质上往往很吵。

0x02 侦察分类

侦察（recon）可分为两部分：

  1. 被动侦察：可以通过被动观看来进行
  2. 主动侦察：需要与目标互动以激发它，以观察其反应。

被动侦察不需要与目标互动。换句话说，您不会向目标或目标拥有的系统发送任何数据包或请求。相反，被动侦察依赖于第三方收集和维护的公开信息。开源情报 (OSINT) 用于收集有关目标的信息，就像查看目标的公开社交媒体资料一样简单。我们可能收集的示例信息包括域名、IP 地址块、电子邮件地址、员工姓名和职位信息。在接下来的任务中，我们将了解如何查询 DNS 记录并扩展被动侦察室的主题，并引入高级工具来帮助您进行侦察。

主动侦察需要通过发送请求和数据包与目标进行交互，并观察其是否响应以及如何响应。收集到的答复（或缺乏答复）将使我们能够扩展我们开始使用被动侦察开发的图景。主动侦察的一个示例是使用 Nmap 扫描目标子网和活动主机。其他示例可以在主动侦察室中找到。我们想要发现的一些信息包括实时主机、正在运行的服务器、监听服务和版本号。

主动侦察可分为：

  1. 外部侦察：在目标网络之外进行，重点关注可通过互联网评估的外部资产。一个例子是从公司网络外部运行 Nikto。
  2. 内部侦察：从目标公司的网络内部进行。换句话说，渗透测试人员或红队测试人员可能实际位于公司大楼内。在这种情况下，他们可能正在目标网络上使用被利用的主机。一个例子是使用 Nessus 使用目标计算机之一扫描内部网络。

0x03 内置工具

这项任务的重点是：

  • whois
  • dig, nslookup, host
  • traceroute/tracert

在开始使用 whois 工具之前，让我们先了解一下 WHOIS。 WHOIS 是遵循 RFC 3912 规范的请求和响应协议。 WHOIS 服务器在 TCP 端口 43 上侦听传入请求。域名注册商负责维护其租赁域名的 WHOIS 记录。 whois 将查询 WHOIS 服务器以提供所有已保存的记录。在下面的例子中，我们可以看到 whois 为我们提供了：

  • Record update date 记录更新日期
  • Registrant contact info and address (unless withheld for privacy)
    注册人联系信息和地址（除非出于隐私原因而保留）
  • Admin contact info and address (unless withheld for privacy)
    管理员联系信息和地址（除非出于隐私原因而保留）

总而言之，我们始终可以信赖：

  • whois 查询 WHOIS 数据库
  • nslookup 、 dig 或 host 用于查询 DNS 服务器

WHOIS 数据库和 DNS 服务器保存公开信息，查询任何一个都不会产生任何可疑流量。

此外，我们可以依靠 Traceroute（Linux 和 macOS 系统上的 traceroute 和 MS Windows 系统上的 tracert ）来发现我们的系统和目标主机之间的跃点。

0x04 高级搜索

能够有效地使用搜索引擎是一项至关重要的技能。下表显示了一些适用于许多流行搜索引擎的流行搜索修饰符。

Symbol / Syntax Function
"search phrase" 查找具有精确搜索短语的结果
OSINT filetype:pdf 查找与特定术语相关的 PDF 类型的文件。
salary site:blog.tryhackme.com 将搜索结果限制在特定站点。
pentest -site:example.com 从结果中排除特定网站
walkthrough intitle:TryHackMe 查找页面标题中包含特定术语的页面。
challenge inurl:tryhackme 查找页面 URL 中包含特定术语的页面。

注意：除了 pdf 之外，需要考虑的其他文件类型还有： doc 、 docx 、 ppt 、 pptx 、 xls 和 xlsx 。

每个搜索引擎可能有一组略有不同的规则和语法。要了解不同搜索引擎的具体语法，您需要访问它们各自的帮助页面。一些搜索引擎（例如 Google）提供了用于高级搜索的 Web 界面：Google 高级搜索。其他时候，最好熟记语法，例如 Google Refine Web Searches、DuckDuckGo Search Syntax 和 Bing Advanced Search Options。

搜索引擎日夜爬行万维网以索引新的网页和文件。有时这可能会导致对机密信息建立索引。机密信息的示例包括：

  • 公司内部使用的文件
  • 包含用户名、电子邮件地址甚至密码的机密电子表格
  • 包含用户名的文件
  • 敏感目录
  • 服务版本号（其中一些可能存在漏洞且未修补）
  • Error messages 错误信息

将高级 Google 搜索与特定术语相结合，可以找到包含敏感信息或易受攻击的网络服务器的文档。谷歌黑客数据库（GHDB）等网站收集此类搜索词并公开可用。让我们看一下 GHDB 的一些查询，看看我们的客户是否有通过搜索引擎暴露的任何机密信息。 GHDB 包含以下类别的查询：

  • Footholds 立足点
    考虑 GHDB-ID: 6364，因为它使用查询 intitle:"index of" "nginx.log" 来发现 Nginx 日志，并可能揭示可被利用的服务器配置错误。
  • Files Containing Usernames 包含用户名的文件
    例如，GHDB-ID: 7047 使用搜索词 intitle:"index of" "contacts.txt" 来发现泄露重要信息的文件。
  • Sensitive Directories 敏感目录
    例如，考虑 GHDB-ID：6768，它使用搜索词 inurl:/certs/server.key 来查明 RSA 私钥是否公开。
  • Web Server Detection 网络服务器检测
    考虑 GHDB-ID: 6876，它使用查询 intitle:"GlassFish Server - Server Running" 检测 GlassFish Server 信息。
  • Vulnerable Files 易受攻击的文件
    例如，我们可以尝试使用查询 intitle:"index of" "*.php" 来查找 PHP 文件，如 GHDB-ID: 7786 提供的那样。
  • Vulnerable Servers 易受攻击的服务器
    例如，要发现 SolarWinds Orion Web 控制台，GHDB-ID: 6728 使用查询 intext:"user name" intext:"orion core" -solarwinds.com 。
  • Error Messages 错误信息
    可以从错误消息中提取大量有用的信息。一个示例是 GHDB-ID: 5963，它使用查询 intitle:"index of" errors.log 来查找与错误相关的日志文件。

您可能需要调整这些 Google 查询以满足您的需求，因为查询将从所有符合条件并已编入索引的网络服务器返回结果。为了避免法律问题，最好不要访问法律协议范围之外的任何文件。

我们建议您加入 Google Dorking 房间以获取更深入的信息。

现在我们将探索两个额外的来源，它们可以在不与我们的目标交互的情况下提供有价值的信息：

  • Social Media 社交媒体
  • Job ads 招聘广告

社交媒体

社交媒体网站已变得非常流行，不仅适合个人使用，而且也适合企业使用。一些社交媒体平台可以透露有关目标的大量信息。尤其如此，因为许多用户倾向于过度分享有关自己及其工作的详细信息。仅举几例，值得检查以下内容：

  • LinkedIn
  • Twitter
  • Facebook
  • Instagram

社交媒体网站可以轻松收集特定公司员工的姓名；此外，在某些情况下，您可能会了解到特定的信息，这些信息可以揭示密码恢复问题的答案或获得包含在目标单词列表中的想法。技术人员的帖子可能会透露有关公司系统和供应商的详细信息。例如，最近获得瞻博网络认证的网络工程师可能会提到在其雇主环境中使用瞻博网络网络基础设施。

招聘广告

招聘广告还可以告诉您很多有关公司的信息。除了透露姓名和电子邮件地址之外，技术职位的职位发布还可以让人们深入了解目标公司的系统和基础设施。不同国家/地区的热门职位可能有所不同。请务必检查您的客户将发布广告的国家/地区的职位列表网站。此外，始终值得检查他们的网站是否有任何职位空缺，并看看这是否会泄露任何有趣的信息。

请注意，Wayback Machine 有助于检索客户网站上职位空缺页面的先前版本。

0x05 专业搜索引擎

WHOIS 和 DNS 相关

除了我们在任务 3 中介绍的标准 WHOIS 和 DNS 查询工具之外，还有第三方为历史 WHOIS 数据提供付费服务。一个例子是 WHOIS 历史记录，它提供了 WHOIS 数据的历史记录，如果域名注册者在注册域名时没有使用 WHOIS 隐私，它会派上用场。

有一些网站提供免费的高级 DNS 服务。其中一些网站提供丰富的功能，并且可以拥有一个完整的房间专门用于探索一个领域。现在，我们将重点关注与 DNS 相关的关键方面。我们将考虑以下事项：

  • ViewDNS.info
  • Threat Intelligence Platform

ViewDNS.info

ViewDNS.info 提供反向 IP 查找。最初，每个 Web 服务器将使用一个或多个 IP 地址；然而，如今，共享托管服务器很常见。通过共享主机，一个 IP 地址可以在具有不同域名的许多不同 Web 服务器之间共享。通过反向 IP 查找，您可以从域名或 IP 地址开始，使用特定 IP 地址查找其他域名。

在下图中，我们使用反向 IP 查找来查找与 cafe.thmredteam.com 使用相同 IP 地址的其他服务器。因此，需要注意的是，知道 IP 地址并不一定会指向单个网站。

Threat Intelligence Platform

威胁情报平台要求您提供域名或IP地址，它将启动从恶意软件检查到WHOIS和DNS查询的一系列测试。 WHOIS 和 DNS 结果与我们使用 whois 和 dig 获得的结果类似，但威胁情报平台以更具可读性和视觉吸引力的方式呈现它们。我们从报告中获得了额外的信息。例如，在查找 thmredteam.com 后，我们看到名称服务器 (NS) 记录已解析为其各自的 IPv4 和 IPv6 地址，如下图所示。

另一方面，当我们搜索 cafe.thmredteam.com 时，我们还可以获得同一 IP 地址上的其他域的列表。我们在下图中看到的结果与我们使用 ViewDNS.info 获得的结果类似。

0x06 Recon-ng

Recon-ng 是一个有助于自动化 OSINT 工作的框架。它使用来自不同作者的模块并提供多种功能。有些模块需要按键才能工作；该密钥允许模块查询相关的在线API。在此任务中，我们将演示在终端中使用 Recon-ng。

从渗透测试和红队的角度来看，Recon-ng 可用于查找有助于操作或 OSINT 任务的各种信息片段。所有收集的数据都会自动保存在与您的工作空间相关的数据库中。例如，您可能会发现主机地址以供稍后进行端口扫描或收集联系人电子邮件地址以进行网络钓鱼攻击。

您可以通过运行命令 recon-ng 来启动 Recon-ng 。启动 Recon-ng 会给你一个类似 [recon-ng][default] > 的提示。在此阶段，您需要选择要使用的已安装模块。但是，如果这是您第一次运行 recon-ng ，您将需要安装所需的模块。

在此任务中，我们将遵循以下工作流程：

  1. 为您的项目创建一个工作区
  2. 将起始信息插入数据库
  3. 在市场中搜索模块并在安装前了解它
  4. 列出已安装的模块并加载一个
  5. 运行加载的模块

创建工作空间

运行 workspaces create WORKSPACE_NAME 为您的调查创建一个新工作区。例如， workspaces create thmredteam 将创建一个名为 thmredteam 的工作区。

recon-ng -w WORKSPACE_NAME 开始使用特定工作区进行 recon-ng。

为数据库播种

在侦察中，您从一条信息开始，然后将其转换为新的信息。例如，您可以从公司名称开始研究，并使用它来发现域名、联系人和个人资料。然后，您将使用获得的新信息对其进行进一步转换并了解有关您的目标的更多信息。

让我们考虑一下这样的情况：我们知道目标的域名 thmredteam.com ，并且我们希望将其输入到与活动工作区相关的 Recon-ng 数据库中。如果我们想检查数据库中表的名称，我们可以运行 db schema 。

我们要将域名 thmredteam.com 插入域表中。我们可以使用命令 db insert domains 来做到这一点。

0x07 Maltego

Maltego 是一款将思维导图与 OSINT 相结合的应用程序。一般来说，你会从域名、公司名称、人名、电子邮件地址等开始。然后你可以让这条信息经历各种转换。

Maltego 中收集的信息可用于后续阶段。例如，收集的公司信息、联系人姓名和电子邮件地址可用于创建看起来非常合法的网络钓鱼电子邮件。

将 Maltego 图上的每个块视为一个实体。一个实体可以有描述它的值。在 Maltego 的术语中，转换是一段代码，用于查询 API 以检索与特定实体相关的信息。逻辑如下图所示。与实体相关的信息通过转换返回零个或多个实体。

值得一提的是，Maltego 中可用的一些转换可能会主动连接到目标系统。因此，如果您想将自己限制为被动侦察，最好在使用变换之前了解其工作原理。

每一次转变都可能产生几个新的值。例如，如果我们从“DNS 名称” cafe.thmredteam.com 开始，我们期望根据我们使用的转换获得新类型的实体。例如，“To IP Address”预计返回 IP 地址，如下所示。

在 Maltego 上实现此目的的一种方法是右键单击“DNS 名称” cafe.thmredteam.com 并选择：

  1. Standard Transforms 标准变换
  2. Resolve to IP 解析为IP
  3. To IP Address (DNS) 至 IP 地址 (DNS)

执行此转换后，我们将获得一个或多个 IP 地址，如下所示。

然后我们可以选择对其中一个 IP 地址应用另一种转换。考虑以下变换：

  1. DNS from IP 从 IP 获取 DNS
  2. To DNS Name from passive DNS (Robtex)
    从被动 DNS (Robtex) 到 DNS 名称

此转换将使用新的 DNS 名称填充我们的图表。再点击几下，您就可以获得 IP 地址的位置等等。结果可能类似于下图。

上述两个示例应该让您了解使用 Maltego 的工作流程。您可以观察到所有工作都基于转换，Maltego 将帮助您保持图表井井有条。通过查询不同的在线网站和数据库，您会得到相同的结果；然而，Maltego 只需点击几下即可帮助您获取所需的所有信息。

我们在之前的任务中尝试了 whois 和 nslookup 。您可以获得大量信息，从姓名、电子邮件地址到 IP 地址。 whois 和 nslookup 的结果直观地显示在下面的 Maltego 图中。有趣的是，Maltego 转换能够提取并排列从 WHOIS 数据库返回的信息。尽管由于隐私保护，返回的电子邮件地址没有帮助，但值得一看的是 Maltego 如何提取此类信息以及如何呈现这些信息。

既然我们已经了解了 Maltego 的力量如何源自其变换，那么唯一合乎逻辑的事情就是通过添加新的变换来使 Maltego 更加强大。转换通常根据数据类型、定价和目标受众分为不同的类别。尽管许多转换可以通过 Maltego Community Edition 和免费转换来使用，但其他转换需要付费订阅。下面显示了一个屏幕截图，以提供更清晰的想法。

0x08 总结

孙子曾说过：“知己知彼，百战不殆。知己知彼，百胜必有败。若不知敌不知己，则百战必败。”快进到网络战争时代；除了了解我们的红队技能和能力之外，我们还需要获得尽可能多的有关目标的信息。地形在不断发展，新的数据收集方法正在成为可能。
